## 공통 구조

### 상단 헤더
- 가상자산 전송인 (텍스트)
- 가상자산 수취인 (텍스트)

### 탭 전환 (3개)
- **Type A: 암호화폐 → KRW**
- **Type B: KRW → 암호화폐 (거래대금 고정)**
- **Type C: KRW → 암호화폐 (수량 고정)**

### 기준가 확정 체크박스
- 각 행의 기준가 셀 안에 체크박스 존재
- **체크 = 확정**: 흰색 배경, 정상 표시
- **미체크 = 미확정**: 노란 배경(#fff8e1) + 적갈색 글씨(#bf360c)
- 미확정 항목이 있으면 상단 워닝: "노란색 기준가는 예상 수치이며 확정된 가격이 아닙니다"

### 행별 공통 입력 필드
- 거래일: `<input type="date">` (연도 4자리 제한)
- 기준거래소: `<input>` + `<datalist>` (빗썸/업비트/코인원/코빗/바이낸스/크라켄/OKX/Bybit + 직접입력)
- 기준가시간: `<select>` 전일 종가 + 30분 단위 (당일 00:00~23:30), **디폴트: 당일 10:00**

### 거래 가상자산 / 코인 단위
- `<input>` + `<datalist>` (USDT/USDC/BTC/ETH/SOL + 직접입력)
- 컬럼 헤더가 선택한 코인에 따라 동적 변경
- 기준가 페어: `{코인}/KRW` 형태 (코인이 앞)

## Type A — 암호화폐 → KRW

### 수수료 설정 (상단)
- 거래 가상자산, 에스크로율(%), 매매율(%) — 기본값 역할
- "전체 행 적용" 버튼으로 일괄 변경
- 각 행에서 개별 편집 가능 (행별 수수료율)

### 테이블 컬럼 (좌→우 논리 순서)
```
No | 거래일 | 기준거래소 | 기준가시간 | 거래금액({코인}) | 에스크로율(%) | 매매율(%) | 총수수료율 | 총수수료금액 | 에스크로수수료 | 수수료차감{코인}(에스크로) | 매매수수료 | 수수료차감{코인}(전체) | 기준가({코인}/KRW) | 원화수령액 | 검증 | 🗑
```

### 계산 로직
- 총수수료율 = 에스크로율 + 매매율
- 총수수료금액 = 거래금액 × 총수수료율 / 100
- 에스크로수수료 = 거래금액 × 에스크로율 / 100
- 수수료차감(에스크로) = 거래금액 − 에스크로수수료
- 매매수수료 = 거래금액 × 매매율 / 100
- 수수료차감(전체) = 거래금액 − 총수수료금액
- 원화수령액 = 수수료차감(전체) × 기준가 (소수점 절사)
- 검증 = 에스크로수수료 + 매매수수료 (= 총수수료금액이면 초록, 아니면 빨강)

### 합계 행
- 금액 컬럼: SUM
- 수수료율 컬럼: 가중평균 (수수료금액합 / 거래금액합 × 100)

## Type B — KRW → 암호화폐 (거래대금 고정)

### 개념
KRW 거래대금이 정해져 있고, 코인수량을 역산

### 수수료 설정 (상단)
- 코인 단위, 거래수수료율(%), 에스크로율(%) — 기본값 역할
- "전체 행 적용" 버튼
- 각 행 개별 편집 가능

### 테이블 컬럼 (좌→우 논리 순서)
```
No | 거래일 | 기준거래소 | 기준가시간 | 거래대금(KRW) | 기준가({코인}/KRW) | 코인수량 | 거래수수료율(%) | 거래수수료 | WB Prime입금액 | 에스크로율(%) | 에스크로수수료 | 출금가능금액 | 🗑
```

### 계산 로직 (순차 차감)
1. 코인수량 = 거래대금 ÷ 기준가
2. 거래수수료 = 코인수량 × 거래수수료율 / 100
3. WB Prime입금액 = 코인수량 − 거래수수료
4. 에스크로수수료 = WB Prime입금액 × 에스크로율 / 100
5. 출금가능금액 = WB Prime입금액 − 에스크로수수료

## Type C — KRW → 암호화폐 (수량 고정) ★NEW

### 개념
출금 목표 코인수량이 정해져 있고, 필요한 KRW 거래대금을 역산
Type B와의 차이: B는 "원화가 기준", C는 "코인수량이 기준"

### 수수료 설정 (상단, Type B와 동일 구조)
- 코인 단위, 거래수수료율(%), 에스크로율(%)

### 테이블 컬럼 (좌→우 논리 순서)
```
No | 거래일 | 기준거래소 | 기준가시간 | 목표수량({코인}) | 기준가({코인}/KRW) | 필요거래대금(KRW) | 코인수량 | 거래수수료율(%) | 거래수수료 | WB Prime입금액 | 에스크로율(%) | 에스크로수수료 | 출금가능금액(검증) | 🗑
```

### 계산 로직 (역산)
- 필요거래대금(KRW) = 목표수량 ÷ (1 − 거래수수료율/100) ÷ (1 − 에스크로율/100) × 기준가
- 코인수량(내부) = 필요거래대금 ÷ 기준가
- 거래수수료 = 코인수량 × 거래수수료율 / 100
- WB Prime입금액 = 코인수량 − 거래수수료
- 에스크로수수료 = WB Prime입금액 × 에스크로율 / 100
- **출금가능금액 = WB Prime입금액 − 에스크로수수료**
  → 출금가능금액 ≈ 목표수량이면 **초록 검증**, 오차 있으면 빨강

## 기능

### 행 관리
- 행 추가 (+) 버튼
- 행 삭제: 휴지통 SVG 아이콘 (각 행 우측)
- 합계 행 자동 계산 (tfoot 고정)

### 저장 / 초기화
- **저장 버튼**: 탭 바 우측에 배치. localStorage에 현재 상태 저장. 타임스탬프 표시
- **초기화 버튼**: 저장 옆. confirm 팝업 후 모든 입력값 + localStorage 삭제
- 페이지 재방문 시 localStorage에서 자동 복원

### xlsx 다운로드
- SheetJS 사용, 브라우저에서 직접 생성
- **수식 살아있음**: 모든 계산 셀에 Excel 수식 + 캐시값 동시 저장
- `fullCalcOnLoad: true` — Excel 열 때 자동 재계산
- 스테이블코인: `TRUNC(..., 2)`, 원화: `TRUNC(..., 0)` 수식 적용
- 셀 숫자 포맷 지정 (#,##0 / #,##0.00 / #,##0.000000)
- 헤더/합계 행 볼드
- 파일명: `{전송인}_{수취인}_{YYYYMMDD}_에스크로수수료.xlsx`
- 현재 활성 탭 기준 1시트

### 링크 복사 (공유)
- 현재 입력 데이터를 JSON → LZ-string 압축 → URL hash 인코딩
- 클립보드에 plain URL 복사
- **공유 링크로 열면 readonly 모드**:
  - 비활성 탭 DOM에서 완전 제거 (저장한 탭만 표시)
  - 탭 전환/저장/초기화/행추가/행삭제/수수료설정 모두 숨김
  - 모든 input/select 편집 불가
  - 기준가 체크박스 숨기되 노란색 스타일은 유지
  - "공유된 문서입니다 (읽기 전용) — xlsx 다운로드만 가능합니다" 배너 표시
  - xlsx 다운로드만 가능
- **주의**: HTML 파일을 웹서버에 호스팅해야 타인 접근 가능 (로컬 file:// 불가)
